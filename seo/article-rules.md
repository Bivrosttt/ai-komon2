# AI顧問室 SEO記事 制作ルール

更新日: 2026-07-13

このルールは、AI顧問室のSEO記事を「検索エンジン向けの水増し」ではなく、読者が一度で判断・行動できる実務コンテンツとして作るための基準である。文字数や画像数はランキング保証ではなく、検索意図を満たすための制作上の目安として扱う。

## 1. 文字数の考え方

Googleに固定の最低文字数・最適文字数はない。検索意図を満たすために必要な量を優先し、冗長な説明で水増ししない。

AI顧問室では、次の目安を使う。

| 記事タイプ | 日本語本文の目安 | 対象例 |
|---|---:|---|
| 狭い疑問・用語解説 | 2,000〜3,500字 | 「AI導入のメリットとは」 |
| 手順・比較・導入判断 | 4,000〜6,000字 | 「AI導入の進め方」「営業効率化」 |
| 広いテーマの基幹記事 | 6,000〜9,000字 | 「中小企業の業務改善」 |

上位10ページの平均文字数は参考値に留め、平均へ機械的に合わせない。競合にある論点、読者の追加疑問、AI顧問室独自の判断材料を埋めた時点で完成とする。

今回追加した10本は、本文が約1,450〜1,900字であり、狭い疑問への回答としては成立するが、広い導入・比較意図を狙う本命記事としては薄い。再編集時は4,000〜6,000字帯を基本に、重複ではなく「判断基準・具体例・失敗例・確認手順」を増やす。

## 2. 記事の標準構成

1. **タイトル / H1**: 主キーワードと読者の到達点を入れる。
2. **導入**: 誰の何の悩みを解決するかを80〜150字で明示する。
3. **先に結論**: 120〜200字で答え、読者が読む価値を判断できるようにする。
4. **目次**: H2の意図が一覧で分かるようにする。
5. **判断基準**: 何を選び、何を避けるかを最初に示す。
6. **具体的な手順 / 比較 / 事例**: 4〜6個のH2で、表・リスト・手順に分解する。
7. **必須図解**: 記事固有の業務フロー、比較、判断マップ、チェック構造を1点以上入れる。
8. **限界・注意点**: AIに任せない範囲、例外、費用、情報管理を明記する。
9. **無料ツール / CTA**: 読者が記事の結論を試せる導線を1つ置く。
10. **FAQ**: 本文で解決しきれない3〜5問を、実際に見える本文として置く。
11. **まとめ / 次に読む**: 次の行動と関連記事を示す。
12. **出典・更新日**: 公開情報、取得日、更新日を残す。

### 記事下部の共通CV

SEO記事の下部に置く主CTAは、記事テーマ別のツール・診断・デモではなく、全記事でAI顧問室ホームページのサービス案内へ統一する。記事本文中の関連記事リンクや補助ツールへのリンクは残してよいが、下部のCVは次の同一ブロックを使う。

```html
<div class="tool-cta service-cta"><h2>AI導入の相談はサービスから</h2><p>AIに任せる業務の選定から、実装、社内ルール、現場への定着までを一気通貫で支援します。自社でどこから始めるか、サービス内容を確認してください。</p><a class="button" href="../../#service">AI顧問室のサービスを見る</a></div>
```

`../../#service` は、公開サーバーとローカルの`file://`プレビューの両方で記事からホームページへ戻れる相対リンクである。CTAはFAQ・出典・関連記事より前に置き、記事ごとに文言や遷移先を変えない。

各H2は、冒頭で結論を答え、その後に理由・具体例・注意点を置く。H2直下の回答を40〜80字で固定するのではなく、質問の複雑さに応じて120〜250字まで使う。

## 3. 図解・画像の必須ルール

### 必須条件

- SEO記事1本につき、**記事固有のAI生成図解を最低1点**入れる。
- 図解は記事の主張を補助するものにする。汎用的な会議写真や飾り画像だけでは不可。
- 図解は**テキスト入り**を標準とする。工程名、判断軸、分類名、短い結論などを画像内に生成し、画像だけでも構造が伝わるようにする。
- 配置は、結論ボックスの直後または最初の重要H2の直後。本文の内容と近接させる。
- 形式はWebPまたはPNG、原則16:9、幅1600px以上。LCP画像は軽量化し、本文内画像はlazy-loadする。
- ファイル名は`<slug>-diagram.webp`のように意味を持たせる。
- `img`要素、説明的な`alt`、`figcaption`を必ず付ける。CSSの背景画像だけで済ませない。

### 生成と正確性の分担

画像生成AIには、構図・色・業務フローに加え、短い日本語ラベル、工程名、分類名、判断軸を生成させる。プロンプトには画像内へ入れる文言を「完全一致」で列挙し、長い文章や未確認の数値は入れない。生成後は、文字化け、欠落、誤字、矢印との重なり、スマホ表示での可読性を目視確認する。

画像内テキストだけに重要情報を依存させない。法的・契約的に重要な条件、詳細な手順、数値の定義、長い説明はHTML/SVG、表、本文、figcaptionにも残す。画像のラベルは理解を早めるための要約として扱う。

図解には次を添える。

- 何を表す図か分かる`alt`
- 図から読み取るべき結論を一文で書く`figcaption`
- 本文から図へ自然につながる前置き
- 画像だけで意味が欠けない本文側のテキスト説明
- 画像内ラベルの完全一致リストと、生成後の可読性確認

### 図解の選び方

| 検索意図 | 推奨図解 |
|---|---|
| 手順・進め方 | ステップフロー / 30日ロードマップ |
| 比較・選び方 | 2軸マップ / 比較表の補助図 |
| リスク・ルール | 入力→処理→出力のリスク層 / 判断フロー |
| 業務効率化 | 現状→改善後のBefore / After |
| ROI・費用対効果 | 時間→金額→回収期間の計算フロー |
| 事例紹介 | 困りごと→AI補助→人の確認→実行の業務接続図 |

### 画像のSEO上の位置づけ

図解は文字数の代わりではない。本文の理解、滞在中の判断、画像検索への発見性を補助する。画像を入れたから順位が上がるとは書かない。Googleは関連テキストの近くに画像を置き、説明的なファイル名とaltを付けることを推奨している。

## 4. 根拠と独自性

- 公開情報の主張には、一次情報または信頼できる資料へのリンクを付ける。
- 数字は取得日・対象・定義を確認し、確認できない数字は書かない。
- AI顧問室の実績でない架空例は、必ず「想定例」「サンプル」と明記する。
- 既存記事の言い換えではなく、判断表、導入前チェック、失敗条件、実装上の注意を追加する。
- 記事の主キーワードは1つに固定し、関連語は検索意図を補う範囲で使う。

## 5. FAQと構造化データ

FAQは読者向けの本文として残す。ただし、GoogleはFAQリッチリザルトの表示を2026-05-07以降終了しているため、FAQPage JSON-LDを順位や表示の効果として期待しない。構造化データは可視本文と一致する場合だけ使い、Article / Breadcrumbを優先する。

## 6. 公開前チェック

- 主キーワードの検索意図を1文で説明できる。
- 文字数は目安を満たし、不要な水増しがない。
- 結論、判断基準、具体例、限界、次の行動がある。
- 記事固有のAI生成図解が1点以上あり、`img`・alt・figcaptionがある。
- 画像内の短い日本語ラベルが正確で読みやすい。長い説明・重要な数値・条件はHTML/SVGでも補っている。
- 内部リンク3〜8本、一次情報への外部リンク2〜5本がある。
- title / description / canonical / OGP / H1 / Article / Breadcrumbを確認する。
- モバイル表示、画像の読み込み、リンク切れを確認する。
- Search Consoleで公開後の表示回数、CTR、検索語、問い合わせ導線を計測する。

## 7. 検索上位記事の見た目・図解リサーチ

記事を書く前に、主キーワードの検索上位候補を2〜5ページ確認する。順位は日時・地域・端末で変わるため、固定順位ではなく取得日時つきの観測として扱う。ファーストビューと全体ページをスクリーンショットで保存し、次を記録する。

- H1までの順序: カテゴリ、パンくず、公開日、著者、カバー画像、結論、要点箱
- 目次の形式: 箇条書き、カード、ロードマップ、ジャンプリンク
- 図解の役割: 手順、比較、判断、分類、Before/After、計算、事例のどれか
- 図解の構造: ノード数、矢印、短いラベル、色分け、人の確認点、例外分岐
- 本文との関係: 画像の直前・直後の説明、表や箇条書きとの重複、figcaption
- 信頼要素: 著者・監修者、更新日、一次情報、事例の範囲、注意書き
- CTAの位置: 結論前に割り込むか、要点を渡した後か、記事テーマと一致しているか

競合から借りるのは、読者が理解しやすい情報の順序と視覚リズムだけにする。見出し、本文、画像、ロゴ、配色、HTML/CSSをコピーしない。AI顧問室では、既存のネイビーとゴールドを基調に、短い日本語ラベルを含む記事固有図解、結論ボックス、要点箱、目次カードを組み合わせる。

検索上位に図解が少ない場合も、装飾画像を増やすのではなく、読者が迷う判断点を1枚の図にする。重要な条件や数値は画像だけに置かず、本文・表・箇条書き・figcaptionにも残す。調査スクショと採用理由は `seo/evidence/competitors/` と日付付き分析メモに保存する。
