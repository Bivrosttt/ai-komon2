CASE 01｜製造業

板金加工業を想定した場合

従業員30名の想定モデル

図面と経験がものを言う現場ほど、業務がベテラン一人に集中しがちです。その属人化をどう崩すかを想定したシナリオです。

よくある状態

見積もりはベテラン一人に集中していて、その人が不在だと止まってしまう。図面から材質・板厚・加工工程を拾い出す作業に時間がかかり、他の仕事が後回しになる。受注前の客先問い合わせへの対応も後手に回りがちで、機会を逃すこともある——というのが、よくある状態です。

90日の動き
1ヶ月目

棚卸しと診断

見積り業務と過去の受注データを棚卸し。材質・板厚・加工工程ごとの単価基準を整理し、AI活用の優先順位を診断します。

2ヶ月目

見積書AIの試験導入

過去の見積もりと単価表を学習させた見積書AIを試験導入。図面情報とメモから見積たたき台を自動作成し、ベテランは確認だけに回れる体制を作ります。

3ヶ月目

定着と次の一手

定型的な客先問い合わせに問い合わせ返信AIを組み合わせ、受注前対応のスピードを底上げ。運用ルールを整え、現場に定着させます。

CASE 02｜建設業

住宅リフォーム業を想定した場合

従業員25名の想定モデル

現場ごとに仕様が異なるリフォーム特有の「ゼロから作る」負担を、どう減らせるかを想定したシナリオです。

よくある状態

現場ごとに仕様が違うため、見積もりのたびにゼロから作っている。現場写真や進捗報告のフォーマットがバラバラで、事務所に戻ってからの取りまとめ作業が重い。お客様への提案資料づくりに時間がかかり、他の商談に手が回らない——というのが、よくある状態です。

90日の動き
1ヶ月目

棚卸しと診断

リフォーム案件のパターン（水回り・外壁・内装など）を洗い出し、共通する見積項目と現場報告の型を診断します。

2ヶ月目

見積書AIの導入

過去の見積もりを学習させた見積書AIを導入。現場調査メモから概算見積のたたき台を自動生成し、営業担当の作成負担を減らします。

3ヶ月目

写真報告書AIで定着

現場写真から日報・報告書を自動作成する写真報告書AIを組み合わせ、事務所での取りまとめ時間を圧縮。運用を定着させます。

業務BeforeAfter

CASE 03｜卸売業

食品卸業を想定した場合

従業員40名の想定モデル

電話・メールに散らばる問い合わせと、埋もれがちな顧客の声。この2つをどう扱えるかを想定したシナリオです。

よくある状態

取引先からの注文変更・問い合わせがメールと電話に混在し、対応漏れが起きやすい。クレームや要望の声が営業担当者の頭の中にしか残らず、商品開発や仕入れ判断に活かせていない。繁忙期は電話対応に追われ、他の業務が止まる——というのが、よくある状態です。

90日の動き
1ヶ月目

棚卸しと診断

問い合わせ・クレームの種類と頻度を棚卸しし、AIで一次対応できるものと人が判断すべきものを診断・仕分けします。

2ヶ月目

問い合わせ返信AIの導入

定型的な注文確認・問い合わせに問い合わせ返信AIを導入し、一次対応のスピードと対応漏れをゼロに近づけます。

3ヶ月目

顧客の声分析AIで定着

蓄積された問い合わせやクレームの記録を顧客の声分析AIでまとめ、傾向を仕入れ・商品開発の会議で使える形に整理。運用を定着させます。

Before → After（想定）
業務BeforeAfter

繁忙期の電話一次対応担当者総出で対応→定型部分はAIが下書き、担当は判断に集中

CASE 04｜サービス業

ハウスクリーニング業を想定した場合

従業員15名の想定モデル

よくある状態

90日の動き
